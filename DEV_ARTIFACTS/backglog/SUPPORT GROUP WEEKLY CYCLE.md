SUPPORT GROUP WEEKLY CYCLE –
COMPLETE CONCEPT
Innerspark support groups operate in structured weekly cycles.
This structure applies to both:
●
●
Brand new groups that are starting for the first time
Existing groups that have already been running
The system is designed to allow continuous enrollment while protecting emotional safety and
group structure.
1. Group Structure
Each support group (for example, Anxiety Support or Work Stress) runs in weekly cohorts.
Every cohort:
●
●
●
●
●
Has a fixed start date (e.g., Monday)
Runs for 7 days
Has a minimum of 2 members required to start
Has a maximum capacity limit is 10
Requires weekly subscription renewal (25,000 UGX per week)
No group allows mid-week entry.
Members can only join at the beginning of a new weekly cycle.
2. Subscription Before Start Date
Before a group starts:
●
●
●
●
It opens for subscription.
Users can browse and select the group.
They see the price (25,000 UGX per week) and the start date.
They pay to reserve a seat.
The group remains open for subscription until:
●
●
It reaches maximum capacity, OR
The official start date arrives.
If the group reaches capacity early, it closes immediately.
3. Minimum Member Rule
On the official start date:
●
●
If there are 2 or more members → the group starts.
If there is only 1 member → the group does not start.
If it does not start:
●
●
The single member is moved to the next cohort, OR
The payment is refunded.
This rule applies to both new groups and existing groups.
4. Weekly Cycle System (Continuous
Flow)
The system runs in overlapping weekly cycles.
Example timeline:
Week A starts on Monday.
Three days before Week A ends (Thursday or Friday):
●
●
●
●
Week B automatically opens for subscription.
New users can start paying to join Week B.
They reserve their seat.
They officially enter on the next Monday.
This pattern continues:
Week A runs
Week B opens before Week A ends
Week B starts
Week C opens before Week B ends
Week C starts
This creates a continuous weekly enrollment system.
5. Existing Groups
If a group category has already been running for months, the same structure applies.
For example:
Anxiety Group is currently running Week 10.
During Week 10:
●
●
●
Week 11 opens three days before Week 10 ends.
New members subscribe for Week 11.
They join officially on the next Monday.
New users never join an already running week.
They always wait for the next start date.
This keeps the emotional journey structured and protected.
6. Weekly Subscription & Renewal
Each member subscribes for one week at a time.
Access lasts 7 days from the start date of the cohort.
When the week ends:
●
●
●
●
●
The subscription expires.
The user becomes inactive.
They cannot send new messages.
They cannot see new messages posted after their expiry.
They can only see messages from when they were active.
To remain in the group, the member must renew for another week.
When they renew:
●
●
Full access is restored.
They can participate again.
7. Access Rules Summary
Active member:
●
●
●
Can send messages
Can see all new messages
Can attend sessions
Expired member:
●
●
●
●
Cannot send messages
Cannot see new messages
Can only see old messages
Can renew anytime
8. Why This Model Is Used
This structured weekly cycle ensures:
●
●
●
●
●
●
Emotional safety (no disruption from mid-week entries)
Stronger group bonding
Predictable therapist scheduling
Continuous revenue flow
Organized enrollment
Scalability
Final Simple Summary
Every support group runs in weekly cycles.
Members:
●
●
●
●
Pay before the start date.
Join only at the beginning of the week.
Cannot join mid-week.
Must renew every week to remain active.
New cohorts open three days before the current week ends.
This applies to both new groups and existing long-running groups.
The system continuously collects members while maintaining structure and safety.
