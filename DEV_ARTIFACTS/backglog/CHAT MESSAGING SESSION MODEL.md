CHAT MESSAGING SESSION MODEL –
STRUCTURED CONCEPT
1. Booking Model Decision
Since therapists are outsourced vendors (not in-house full-time staff), the system must follow a
scheduled booking model, not instant real-time matching.
Therefore:
●
Clients cannot randomly message therapists who are “online.
”
●
Clients must book a chat session based on the therapist’s availability.
●
Chat sessions follow the same availability structure as video therapy.
This ensures:
●
●
Therapist control over schedule
Professional boundaries
●
●
Predictable session timing
No unmanaged demand pressure
2. Client Booking Flow
Step 1: Client Browses Therapists
Client can:
●
View therapist profiles
●
View specialization
●
●
●
View availability calendar
View pricing
Select session type:
○
Video Therapy
○
Chat Messaging Session
Step 2: Client Selects Chat Session
If client selects “Chat Messaging”:
They must:
●
Choose a date
●
Choose a time slot (based on therapist availability)
●
See session duration (e.g., 1 hour)
●
See session price
●
Proceed to payment
Step 3: Payment Confirmation
Once payment is successful:
●
Session is confirmed.
●
Both therapist and client receive confirmation.
●
Chat remains locked until the scheduled time.
The chat does NOT open immediately.
3. Chat Session Activation Logic
At the exact scheduled time:
●
The chat window automatically opens.
●
Both therapist and client can send messages.
●
Session timer begins (e.g., 60 minutes).
Both must be online at the same time to interact.
4. Session Duration Rule
Each chat session has a fixed duration (example: 1 hour).
During that 1 hour:
●
Both can send unlimited messages.
●
Real-time typing.
●
Session timer visible.
When 60 minutes elapse:
●
●
●
Chat automatically closes.
Both parties can no longer send messages.
Conversation becomes read-only.
5. After Session Ends
After closure:
●
Chat remains visible for reference.
●
●
No new messages allowed.
Client must book another session to continue.
Optional future feature:
Allow therapist to extend session manually (if payment processed).
6. Late Arrival Logic
Case 1: Client joins late
●
Timer still ends at scheduled end time.
●
No time compensation unless policy allows.
Case 2: Therapist joins late
●
Timer still runs.
●
Admin may handle complaints or manual extension.
Clear policy needed for fairness.
7. No Instant Chat Model (For Now)
Because therapists are outsourced:
●
No “Therapist Online – Start Chat Now” option.
●
No 24/7 open messaging.
●
No asynchronous therapy.
All chats are scheduled and time-bound.
This avoids:
●
Therapist overload
●
Unpaid labor
●
Blurred boundaries
●
Legal risks
8. Session Control Summary
Client selects therapist →
Chooses Chat Session →
Selects date & time →
Pays →
Waits →
Chat opens at scheduled time →
Timer runs →
Chat closes automatically →
Client books again if needed.
9. Why This Model Is Best for Your Current
Stage
Since therapists are vendors:
●
They control their calendar.
●
You avoid operational chaos.
●
Payment is secured before session.
●
Boundaries are clear.
●
Legal risk is reduced.
When you later build an in-house therapist team, you can introduce:
●
●
●
Instant chat support
On-demand 15-minute micro sessions
Crisis chat model
But not now.
10. Optional Strategic Enhancements
(Future Phase)
You may later add:
●
24-hour text reflection model (client sends long message, therapist responds within 24
hours)
●
Premium priority chat
●
Corporate chat packages
●
AI pre-chat emotional intake form
But your current model should remain structured and scheduled.
Final Concept Summary
Chat Messaging Sessions are scheduled, paid, and time-bound interactions between a client
and a selected therapist.
They:
●
●
●
●
●
Follow therapist availability
Open only at the scheduled time
Run for a fixed duration (e.g., 1 hour)
Close automatically when time ends
Require new booking for continuation
No instant or open-ended chat is allowed under the current outsourced therapist model.
